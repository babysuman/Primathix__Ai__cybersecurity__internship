# \# DVWA: Damn Vulnerable Web Application



#### &#x20; # Definition :

###### &#x20;    DVWA (Damn Vulnerable Web Application) is a PHP/MySQL web application designed for security testing and training purposes. It is intentionally designed with multiple security flaws so that beginners and professionals can practice web penetration testing, ethical hacking, and vulnerability exploitation in a safe environment.



#### \#Command of docker:

###### 1)docker compose down: stop DVWA

###### 2)docker compose up -d: start DVWA again 

###### 3)docker compose logs: to view logs

###### 4)docker compose down -v: to remove everything.



#### \#Why use DVWA?

###### &#x20; Learning tool: Great for students, bug bounty beginners, and security researchers.



###### &#x20; Safe environment: No risk of breaking real websites.

###### &#x20; 

###### &#x20; Skill development: Helps practice both attacking and securing applications.



###### &#x20; Comparison: You can test your exploit on Low/Medium/High security and see how defenses stop it.



#### \#Vulnerabilities We Can Practice

###### &#x20;  DVWA includes almost all OWASP Top 10 security issues. For example:

###### &#x20;1) SQL Injection: Testing how attackers can steal database data.

###### &#x20;2) Command Injection: Running system commands via web input.

###### &#x20;3) File Inclusion: Exploiting Local/Remote File Inclusion (LFI/RFI).

###### &#x20;4) Cross-Site Scripting (XSS): Stealing cookies, injecting malicious scripts.

###### &#x20;5) Brute Force: Cracking weak login credentials.

###### &#x20;6) CSRF (Cross-Site Request Forgery): Forcing users to take unintended actions.

###### &#x20;7) File Upload: Uploading malicious files (like shells).

###### &#x20;8) Weak Session IDs: Exploiting poor session management.
